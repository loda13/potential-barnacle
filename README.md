# 股票K线技术分析工具

命令行股票技术分析工具，支持A股/美股/港股/台股。基于 SMC（聪明钱概念）和 EV（期望值）构建严格决策树，集成多指标共振分析、锚定筹码分布、期权数据、黑天鹅熔断、波动率平价仓位管理和执行日志审计。

## 安装

```bash
pip3 install -r requirements.txt
```

## 使用

```bash
# A股日线（隆基绿能，最近120天）
python3 stock_analyzer.py 601012 -d 120

# 美股（英伟达）
python3 stock_analyzer.py NVDA -d 120

# 港股（腾讯控股）
python3 stock_analyzer.py 0700.HK -d 120

# 批量分析多只股票
python3 stock_analyzer.py --portfolio 0700.HK 1810.HK QQQ TSLA NVDA -d 120

# 压力测试（15% 回撤场景）
python3 stock_analyzer.py NVDA -d 120 --stress-test

# 演示模式（离线，不拉取网络数据）
python3 stock_analyzer.py TSLA --demo
```

## 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `code` | 股票代码（A股6位数字，美股如AAPL，港股如0700.HK，台股如2330.TW） | 必填（单股票模式） |
| `-p, --period` | K线周期：`d`=日线，`w`=周线 | `d` |
| `-d, --days` | 分析天数（建议≥120以获得更可靠的分析） | `60` |
| `--demo` | 演示模式，使用模拟数据 | 关闭 |
| `--portfolio` | 批量分析多只股票 | 无 |
| `--stress-test` | 执行 15% 回撤压力测试 | 关闭 |

## 系统架构

### 决策链（按优先级从高到低）

```
1. 黑天鹅熔断 (Circuit Breaker)     ← 最高优先级，覆写一切
   ├─ 大盘环境阻断: 基准指数跌破 MA200 或 VIX 飙升 >15%
   └─ 个股跳空毁灭: 向下跳空 >1.5x ATR 穿越 HVN

2. SMC 严格决策树                    ← 核心决策引擎
   ├─ 买入 (BUY): 4个条件必须全部满足
   │   ├─ ChoCh 确认 (趋势特性改变)
   │   ├─ 回踩确认 (价格在 HVN 或 FVG 区域)
   │   ├─ R-Multiple ≥ 2.0 (盈亏比)
   │   └─ Z-Score 在 [-2, 2] 区间
   ├─ 卖出 (SELL): 满足任意1个即触发
   │   ├─ 跌破 Chandelier Exit 止损
   │   └─ 派发模式 (放量滞涨)
   └─ 观望 (HOLD): 其他所有情况

3. 波动率平价仓位管理                ← 精确计算建仓量
   ├─ 公式: 股数 = (账户风险额) / (价格 - 止损价)
   ├─ 流动性惩罚: 仓位 ≤ ADV 的 1%
   └─ 滑点复核: 买入+0.2%, 止损-0.2% 后复核 EV

4. 执行日志                          ← 静默追加到 CSV
```

### 分析模块

| 模块 | 功能 |
|------|------|
| **技术指标** | MA, RSI, MACD, 布林带, KDJ, ADX, ATR, OBV, CCI, SuperTrend, PSAR, Ichimoku, VWAP |
| **锚定筹码分布** | 自动识别锚定点（BoS/放量日/跳空），计算 POC/VAH/VAL/HVN |
| **左侧交易** | 均值回归 Z-Score、量能衰竭、波动率状态、三重背离、机构级支撑压力 |
| **右侧交易** | Break of Structure (BoS)、Change of Character (ChoCh) |
| **退出策略** | VWAP 偏离度、高位派发模式识别 |
| **期权数据** | IV/IV Rank/P/C Ratio（美股），假突破检测 |
| **风控** | Chandelier Exit 止损、黑天鹅熔断、VIX 监控 |

### 指标权重设计

- **KDJ / MACD**：降级为辅助指标（`is_auxiliary=True`），不参与主共振计算，仅用于背离检测
- **ADX / SuperTrend / Ichimoku**：主力趋势指标，权重 2
- **其他指标**：标准权重 1

## 全局参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `MAX_RISK_PER_TRADE` | 2% | 每笔交易最大风险比例 |
| `DEFAULT_ACCOUNT_SIZE` | 100,000 | 默认账户规模（元） |
| `CHANDELIER_ATR_MULTIPLIER` | 2.5 | Chandelier Exit ATR 倍数 |
| `CHANDELIER_LOOKBACK` | 22 | Chandelier Exit 回溯天数 |
| `SLIPPAGE_PCT` | 0.2% | 滑点模拟 |
| `ADV_POSITION_LIMIT` | 1% | 仓位不超过 ADV 的比例 |

## 执行日志

每次运行自动追加到项目根目录的 `trading_decision_log.csv`，记录字段：

```
timestamp, code, date, close, atr, iv, poc, adv20,
buy_price, stop_loss, target_price, risk_reward, risk_reward_after_slippage,
decision, action, suggested_shares, circuit_breaker
```

## 常见问题

**Q: 提示"无法获取股票数据"？**
A: 检查网络连接，或使用 `--demo` 模式测试。批量分析时可能触发 Yahoo Finance 频率限制，建议逐个分析。

**Q: A股代码如何输入？**
A: 直接输入6位数字，如 `600519`、`000876`

**Q: 港股如何输入？**
A: 使用 yfinance 格式，如 `0700.HK`（腾讯）、`1810.HK`（小米）

**Q: 为什么系统总是输出"持仓观望"？**
A: SMC 决策树要求 4 个买入条件全部满足才会输出买入信号。最常见的缺失条件是 ChoCh（趋势特性改变），这意味着市场尚未出现明确的趋势反转信号。这是设计上的严格性，避免在震荡市中频繁交易。

**Q: trading_decision_log.csv 在哪里？**
A: 在项目根目录下，每次运行自动追加，无需手动创建。
